# Feature Specification: きのみ情報一覧

**Feature Branch**: `028-berry-info`  
**Created**: 2026-02-03  
**Status**: Draft  
**Input**: User description: "きのみに関するデータを一覧で閲覧したい。きのみごとの基礎エナジーはゲーム上からは閲覧できないため。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - きのみ一覧の閲覧 (Priority: P1)

ユーザーがポケモンスリープのきのみの基礎エナジーを確認できるよう、全てのきのみ情報を一覧表示する。ゲーム内では確認できない基礎エナジーを簡単に参照できることが主な価値。

**Why this priority**: ゲーム内で確認できない基礎エナジー情報へのアクセスが本機能の核心であり、これがないと機能の存在意義がない。

**Independent Test**: きのみページにアクセスし、全きのみの名前、タイプ、基礎エナジーが表示されることを確認できる。

**Acceptance Scenarios**:

1. **Given** ユーザーがきのみページにアクセスした, **When** ページが読み込まれた, **Then** 全てのきのみが一覧表示される
2. **Given** きのみ一覧が表示されている, **When** 各きのみを確認する, **Then** きのみ名、タイプ、基礎エナジー、アイコンが表示されている

---

### Edge Cases

- 画像が読み込めない場合、代替表示（タイプ名のテキストバッジなど）を表示する
- きのみデータが空の場合、適切なメッセージを表示する

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは全てのきのみを一覧表示できなければならない
- **FR-002**: 各きのみについて、きのみ名を表示しなければならない
- **FR-003**: 各きのみについて、対応するポケモンタイプを表示しなければならない
- **FR-004**: 各きのみについて、基礎エナジーを数値で表示しなければならない
- **FR-005**: 各きのみについて、簡易アイコン（タイプを示すビジュアル）を表示しなければならない
- **FR-006**: 一覧はタイプ別に視覚的に区別できなければならない

### Key Entities

- **Berry（きのみ）**: きのみ名、タイプ（ポケモンタイプ）、基礎エナジー（数値）を持つエンティティ。各きのみは1つのポケモンタイプに対応する。

### Assumptions

- ポケモンスリープのきのみは全18種類（各ポケモンタイプに1つ）存在する
- 基礎エナジーは固定値でゲームのアップデートで変更される可能性がある（データは静的に管理）
- アイコンはタイプを示す簡易的な表示（色分けやバッジなど）で十分

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーは1回のページアクセスで全きのみの基礎エナジーを確認できる
- **SC-002**: 各きのみについて、名前・タイプ・基礎エナジー・アイコンの4項目が欠けなく表示される
- **SC-003**: きのみ情報は1秒以内に表示される
