# Feature Specification: ナビゲーションバーアイテムの整理

**Feature Branch**: `005-organize-navbar-items`
**Created**: 2026-02-02
**Status**: Draft
**Input**: User description: "ナビゲーションバーに表示するアイテムの整理。睡眠戦略とゲームメカニクスを削除。島情報をフィールド情報に変更。きのみ情報と食材情報を追加。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 整理されたナビゲーションでコンテンツにアクセス (Priority: P1)

ユーザーがサイトを訪問したとき、ナビゲーションバーには現在アクティブなコンテンツへのリンクのみが表示され、不要なリンク（睡眠戦略、ゲームメカニクス）が削除されていることで、必要な情報に迷わずアクセスできる。

**Why this priority**: ナビゲーションの整理はサイト全体のユーザビリティに直結し、すべてのユーザーに影響する基本的な改善である。

**Independent Test**: ナビゲーションバーを確認し、「睡眠戦略」と「ゲームメカニクス」リンクが表示されないこと、その他のリンクが正常に機能することを確認できる。

**Acceptance Scenarios**:

1. **Given** ユーザーがトップページを訪問している, **When** ナビゲーションバーを確認する, **Then** 「睡眠戦略」リンクが表示されていない
2. **Given** ユーザーがトップページを訪問している, **When** ナビゲーションバーを確認する, **Then** 「ゲームメカニクス」リンクが表示されていない
3. **Given** ユーザーがサイドバーを開いている, **When** メニュー項目を確認する, **Then** 削除対象の項目が表示されていない

---

### User Story 2 - フィールド情報へのナビゲーション (Priority: P1)

ユーザーがフィールド（島）の情報を探しているとき、「フィールド情報」という分かりやすいラベルでナビゲーションからアクセスできる。

**Why this priority**: 既存機能の名称変更であり、ユーザーが目的のコンテンツを見つけやすくする重要な改善。

**Independent Test**: ナビゲーションから「フィールド情報」をクリックし、島/フィールド情報ページに遷移することを確認できる。

**Acceptance Scenarios**:

1. **Given** ユーザーがナビゲーションバーを見ている, **When** 「フィールド情報」リンクをクリックする, **Then** フィールド情報ページ（/islands）に遷移する
2. **Given** ユーザーがモバイルでサイドバーを開いている, **When** 「フィールド情報」をタップする, **Then** フィールド情報ページに遷移しサイドバーが閉じる

---

### User Story 3 - きのみ情報へのアクセス (Priority: P2)

ユーザーがきのみ（ベリー）の情報を探しているとき、ナビゲーションバーから「きのみ情報」に直接アクセスできる。

**Why this priority**: 新規コンテンツへのアクセスポイントを追加することで、ユーザーがきのみ関連情報を見つけやすくなる。

**Independent Test**: ナビゲーションから「きのみ情報」をクリックし、きのみ情報ページに遷移することを確認できる。

**Acceptance Scenarios**:

1. **Given** ユーザーがナビゲーションバーを見ている, **When** 「きのみ情報」リンクをクリックする, **Then** きのみ情報ページに遷移する
2. **Given** きのみ情報ページが存在しない場合, **When** 「きのみ情報」をクリックする, **Then** プレースホルダーページまたは「準備中」表示がされる

---

### User Story 4 - 食材情報へのアクセス (Priority: P2)

ユーザーが食材の情報を探しているとき、ナビゲーションバーから「食材情報」に直接アクセスできる。

**Why this priority**: 新規コンテンツへのアクセスポイントを追加することで、ユーザーが食材関連情報を見つけやすくなる。

**Independent Test**: ナビゲーションから「食材情報」をクリックし、食材情報ページに遷移することを確認できる。

**Acceptance Scenarios**:

1. **Given** ユーザーがナビゲーションバーを見ている, **When** 「食材情報」リンクをクリックする, **Then** 食材情報ページに遷移する
2. **Given** 食材情報ページが存在しない場合, **When** 「食材情報」をクリックする, **Then** プレースホルダーページまたは「準備中」表示がされる

---

### Edge Cases

- 削除された「睡眠戦略」や「ゲームメカニクス」のURLに直接アクセスした場合はどうなるか？ → 404ページまたは適切なリダイレクト
- 新しいページ（きのみ情報、食材情報）のコンテンツが未実装の場合はどうなるか？ → プレースホルダーページを表示

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ナビゲーションバーから「睡眠戦略」リンクを削除すること
- **FR-002**: ナビゲーションバーから「ゲームメカニクス」リンクを削除すること
- **FR-003**: 「島ガイド」のラベルを「フィールド情報」に変更すること（URLパスは/islandsのまま維持）
- **FR-004**: 「きのみ情報」リンクをナビゲーションバーに追加すること
- **FR-005**: 「食材情報」リンクをナビゲーションバーに追加すること
- **FR-006**: デスクトップナビゲーション、モバイルサイドバー両方で変更が反映されること
- **FR-007**: 新規追加されるリンクには適切なアイコンを設定すること

### Navigation Item Changes Summary

| 変更種別 | 項目名 | 対応内容 |
|---------|--------|---------|
| 削除 | 睡眠戦略 | ナビゲーションから削除 |
| 削除 | ゲームメカニクス | ナビゲーションから削除 |
| 変更 | 島ガイド → フィールド情報 | ラベル名を変更（パスは維持） |
| 追加 | きのみ情報 | 新規ナビゲーション項目 |
| 追加 | 食材情報 | 新規ナビゲーション項目 |

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ナビゲーションバーに表示される項目数が7個から6個に減少している
- **SC-002**: ユーザーが「フィールド情報」から既存の島情報ページにアクセスできる
- **SC-003**: ユーザーが「きのみ情報」「食材情報」から対応ページにアクセスできる
- **SC-004**: すべてのナビゲーションリンクが正常に動作し、404エラーが発生しない
- **SC-005**: モバイル・デスクトップ両方でナビゲーションの変更が正しく表示される

## Assumptions

- きのみ情報、食材情報の詳細ページコンテンツは本仕様のスコープ外とし、ナビゲーションリンクの追加のみを対象とする
- 新規ページはプレースホルダーページとして最小限の実装を行う
- 削除される「睡眠戦略」「ゲームメカニクス」ページへの直接アクセスは本仕様のスコープ外とする
- きのみ情報のパスは `/berries`、食材情報のパスは `/ingredients` を想定
