# Feature Specification: 食材情報閲覧

**Feature Branch**: `006-ingredient-list`
**Created**: 2026-02-03
**Status**: Draft
**Input**: User description: "ユーザーは食材ごとのエナジーを知りたい。なぜならゲーム上で確認する術がなく、追加食材として投入するときに何を選べばいいのか迷うため"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 食材一覧の閲覧 (Priority: P1)

ユーザーはナビゲーションメニューから食材一覧画面にアクセスし、全ての利用可能な食材とそのエナジー値を確認できる。これにより、レシピに追加する食材を選ぶ際の判断材料を得られる。

**Why this priority**: ゲーム内で食材のエナジー値を確認する手段がないため、これが最も基本的かつ必要な機能である。ユーザーがゲームプレイ中に最適な食材を選択できるようになる。

**Independent Test**: ナビゲーションから食材一覧画面を開き、食材名とエナジー値が表示されることを確認する

**Acceptance Scenarios**:

1. **Given** ユーザーがアプリケーションのホームページにいる, **When** ナビゲーションメニューの「食材」リンクをクリックする, **Then** 食材一覧画面が表示される
2. **Given** ユーザーが食材一覧画面にいる, **When** 画面を閲覧する, **Then** 全ての利用可能な食材がリスト形式で表示される
3. **Given** ユーザーが食材一覧画面にいる, **When** 各食材エントリーを確認する, **Then** 食材アイコン、食材名、基本エナジー値が表示される

---

### User Story 2 - 食材の素早い確認 (Priority: P2)

ユーザーはレシピ作成中に素早く食材のエナジー値を確認したい。一覧が見やすく整理されていることで、必要な情報にすぐアクセスできる。

**Why this priority**: 基本的な一覧表示が実現した後、ユーザビリティを向上させる付加価値機能である

**Independent Test**: 食材一覧画面で特定の食材を探し、エナジー値を確認できることをテストする

**Acceptance Scenarios**:

1. **Given** ユーザーが食材一覧画面にいる, **When** 特定の食材を探す, **Then** 視認しやすいレイアウトで食材情報が表示される

---

### Edge Cases

- 食材データが空の場合: 適切なメッセージを表示する
- 画像アイコンが読み込めない場合: 代替表示（プレースホルダー）を使用する

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、ナビゲーションメニューに「食材」へのリンクを表示しなければならない
- **FR-002**: システムは、食材一覧画面において全ての利用可能な食材を表示しなければならない
- **FR-003**: システムは、各食材エントリーにおいてアイコン、食材名、基本エナジー値を表示しなければならない
- **FR-004**: システムは、食材一覧をユーザーが見やすい形式（リストまたはグリッド）で表示しなければならない
- **FR-005**: システムは、ホームページまたはナビゲーションメニューから食材一覧画面へのアクセスを提供しなければならない

### Key Entities

- **食材 (Ingredient)**: ゲーム内で料理に使用できる材料。食材名、アイコン画像、基本エナジー値を持つ
- **エナジー (Energy)**: 食材が持つ数値。料理の効果を計算するために使用される

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーはナビゲーションから2クリック以内で食材一覧画面にアクセスできる
- **SC-002**: 全ての食材（現時点で存在する全食材）がリストに表示される
- **SC-003**: 各食材エントリーにアイコン、名前、エナジー値の3つの情報が表示される
- **SC-004**: 食材一覧画面は1秒以内に表示が完了する

## Assumptions

- 食材データはアプリケーション内で既に管理されているか、データソースが存在する
- 食材アイコンは画像ファイルとして利用可能である
- エナジー値は数値として保持されている
